# Compose UI Testing Lab

Учебный проект для практики автоматизированного UI-тестирования Android-приложений на Jetpack Compose.

Проект содержит два экрана с различной бизнес-логикой и набором UI-тестов, покрывающих основные пользовательские сценарии.

## Технологии

- Kotlin
- Jetpack Compose
- Material 3
- Compose UI Test
- JUnit4
- Android SDK 36

## Функциональность

### Первый экран

Экран демонстрирует базовую работу со state в Compose.

Возможности:

- Авторизация/выход пользователя
- Счётчик
- Защита от ухода в отрицательные значения
- Отображение ошибок
- Навигация на второй экран

### Второй экран

Мини-приложение для управления списком задач.

Возможности:

- Добавление задач
- Удаление задач
- Отметка задачи как выполненной
- Подсчёт активных задач
- Фильтрация списка
    - Все задачи
    - Только активные
- Пустое состояние списка
- Навигация на первый экран

## Структура проекта

```
App
├── FirstScreen
│   ├── Авторизация
│   ├── Счётчик
│   └── Обработка ошибок
│
└── SecondScreen
    ├── Todo List
    ├── Фильтрация
    ├── Счётчик активных задач
    └── Управление состоянием задач
```

## UI-тесты

Для экранов реализован набор автоматизированных тестов.

### FirstScreenTests

Покрытые сценарии:

- Отображение начального состояния
- Смена статуса пользователя
- Увеличение счётчика
- Уменьшение счётчика
- Проверка ошибки при попытке уйти в минус
- Проверка навигации
- Комбинированный пользовательский сценарий

### SecondScreenTests

Покрытые сценарии:

- Начальное состояние интерфейса
- Пустой список задач
- Добавление задачи
- Удаление задачи
- Очистка поля ввода
- Добавление нескольких задач
- Перевод задачи в статус DONE
- Возврат задачи в ACTIVE
- Подсчёт активных задач
- Работа фильтра "Все"
- Работа фильтра "Активные"
- Возврат к отображению всех задач
- Удаление задач из отфильтрованного списка
- Корректное обновление списка после удаления
- Отображение пустого состояния после фильтрации
- Сохранение данных при переключении фильтров
- Проверка пересчёта активных задач
- Проверка навигации между экранами

## Результат

В рамках проекта было реализовано:

- Более 20 UI-тестов
- Более 500 строк кода
- Использование Test Tags для стабильного поиска элементов
- Работа с состоянием Compose
- Тестирование пользовательских сценариев
- Тестирование фильтрации и динамического контента

## Запуск тестов

Запуск всех UI-тестов:

```bash
./gradlew connectedAndroidTest
```

Или через Android Studio:

```text
Run → All Tests
```
